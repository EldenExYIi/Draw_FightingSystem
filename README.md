# Fight 002 MVP

武器槽位 + 手动打牌战斗系统验证原型。

## 玩法验证点

- 双方自动抽牌，玩家手动选择打出或跳过。
- 抽到牌后会立刻进入下一张牌的抽牌读条；读条结束前未选择则自动打出当前牌。
- 动作牌进入武器槽，槽满后按顺序触发，最后执行武器攻击。
- 法术牌立即触发并进入弃牌堆。
- 抽牌堆、弃牌堆可点击查看牌堆内容。
- 日志中的卡牌名可点击查看效果说明。
- 关键词、状态可点击查看说明。
- 支持实时速度与回合制模式。

## 快捷键

- `1`：0.1x
- `2`：0.5x
- `3`：1x
- `4`：2x
- `5`：3x
- `6`：5x
- `0`：回合制
- `Space`：暂停/继续
- `R`：新战斗

## 数据表

策划数据存放在 `data/` 目录：

- `cards.xlsx`：卡牌数据
- `characters.xlsx`：角色与卡组数据
- `weapons.xlsx`：武器数据
- `keywords.xlsx`：关键词说明
- `battle_config.xlsx`：战斗基础配置

页面默认读取 `data/game_data.js`。如需重新生成表格和页面数据镜像，可运行：

```bash
python3 tools/build_data.py
```

## 本地运行

直接用浏览器打开 `index.html` 即可运行。

也可以启动任意静态服务器，例如：

```bash
python3 -m http.server 8000
```

然后访问 `http://localhost:8000`。

## GitHub Pages

这是纯静态项目，部署到 GitHub Pages 时将 Pages 来源设置为仓库根目录即可。
