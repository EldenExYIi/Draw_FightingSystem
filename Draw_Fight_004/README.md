# Draw Fight 004 MVP

004 验证的是“抽牌策略 + 自动封装牌链”。

## 核心规则

- 双方自动抽牌。
- 玩家不再逐张手动选择编入或弃置。
- 当前抽牌策略会自动决定抽到的牌如何处理。
- 被编入的牌进入待封装区，不会立即触发。
- 每个角色拥有自己的封装规则。
- 满足封装规则后，待封装区自动生成牌链。
- 牌链进入工作区后，按顺序循环触发。

## 当前可验证内容

- 抽牌策略差异：稳步推进、寻找攻击、寻找燃烧组件、防守优先等。
- 封装规则差异：槽位封装、费用封装。
- 角色差异：
  - 燃烧工匠：费用达到阈值后自动封装。
  - 剑术学徒：三槽武器填满后自动封装。
  - 盾链守卫、毒雾术士等用于观察不同规则节奏。
- 工作区牌链固定显示，并独立根据牌链数据循环触发。

## 数据表

策划数据在 `data/` 目录中：

- `cards.xlsx`：卡牌基础数据、费用、打牌时间、标签、效果。
- `strategies.xlsx`：抽牌策略。
- `packaging_rules.xlsx`：封装规则。
- `characters.xlsx`：角色血量、卡组、默认策略、封装规则。
- `keywords.xlsx`：关键词说明。
- `battle_config.xlsx`：抽牌、洗牌、状态结算等基础配置。

修改 `tools/build_data.py` 后，使用项目的 Python 环境重新生成表格和 `game_data.js`。

## 操作

- 点击顶部速度按钮调整播放速度。
- 空格：暂停/继续。
- `1` 到 `6`：切换速度。
- `R`：新战斗。
- 玩家头像下方可以切换当前抽牌策略。

## 本地运行

直接打开 `index.html`，或从根目录版本菜单进入。
